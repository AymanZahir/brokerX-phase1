# ADR-001 — Style d’architecture : Monolithe Hexagonal (Ports & Adapters)

**Statut**: Accepté

## Contexte
Nous devons livrer un prototype clair, testable et évolutif. Séparer le métier des frameworks (web/ORM) facilite les tests et prépare l’évolution (remplacement d’adapters).

## Décision
Adopter un **monolithe hexagonal** avec 3 couches :
- **Domain** : règles métier pures (`Account`, `Wallet`, `Order`, `PreTradeRules`)
- **Application** : cas d’utilisation (`Authenticate`, `DepositFunds`, `PlaceOrder`)
- **Adapters** : Web (REST), Persistance (Repositories/ORM), intégrations simulées (paiement/market)

**Règle de dépendances** : `Adapters → Application → Domain` (pas de cycles).

## Conséquences
- Tests unitaires du métier sans infra
- Remplaçabilité des adapters (web/ORM)
- Trajectoire vers une archi plus distribuée plus tard
- Besoin de mappers/DTO et ports explicites (légère surcharge)
